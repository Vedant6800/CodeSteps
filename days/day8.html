What are HTML Frames?

In the early days of the web, HTML Frames were used to divide a browser window into multiple independent sections. Each section (frame) could load a separate HTML file. This allowed navigation menus to stay fixed while only the content area changed.

Frames were introduced in HTML 4.0 using the <frameset> tag. However, frames are deprecated in HTML5 — modern web development uses CSS Grid, Flexbox, and iframes for similar results.

⚠️
Deprecated: The <frameset> and <frame> tags were removed in HTML5. They are covered here for historical understanding. Use CSS layout and <iframe> for modern projects.

Key Frame Tags

  • <frameset> — Replaces <body>; defines the frame grid using rows and columns
  • <frame> — Defines an individual frame and links it to an HTML file via src
  • <noframes> — Fallback content for browsers that don't support frames
  • <iframe> — Inline frame embedded inside normal page body (still valid in HTML5)
🪟

Classic Frame-Based Layout

A typical frame-based layout divided the browser window into four zones:

  • Header Frame — Top area with site logo and title (remained fixed)
  • Left/Navigation Frame — Sidebar with links; clicking a link changed only the content frame
  • Right/Content Frame — Main area where actual page content loaded
  • Status Frame — Bottom bar showing status, copyright, or notifications

Frameset Code Example

A frameset layout requires a main index file (which uses <frameset> instead of <body>) and separate HTML files for each frame.

HTML — index.html (Main Frameset) 
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN">
<html>
<head>
  <title>My Framed Website</title>
</head>

<!-- rows: 15% header, flexible middle, 10% status -->
<frameset rows="15%,*,10%">

  <!-- Header Frame -->
  <frame src="header.html" name="header"
         scrolling="no" noresize>

  <!-- Middle row: nav + content side by side -->
  <frameset cols="20%,*">
    <frame src="nav.html"    name="nav"     scrolling="no">
    <frame src="content.html" name="content" scrolling="auto">
  </frameset>

  <!-- Status Frame -->
  <frame src="status.html" name="status"
         scrolling="no" noresize>

  <!-- Fallback for browsers without frame support -->
  <noframes>
    <p>Your browser does not support frames.</p>
  </noframes>

</frameset>
</html>
HTML — nav.html (Navigation frame with target) 
<body>
  <h3>Navigation</h3>

  <!-- target="content" loads the page in the content frame -->
  <a href="home.html"    target="content">Home</a>
  <a href="about.html"   target="content">About</a>
  <a href="contact.html" target="content">Contact</a>
</body>

Key Frame Attributes

  • src — HTML file to load in the frame
  • name — identifier used by links' target attribute to direct page loads
  • scrolling="yes|no|auto" — controls scrollbar behaviour
  • noresize — prevents user from resizing the frame border
  • rows / cols on <frameset> — defines the grid in pixels, percent, or * (remaining space)
🚀

Modern Alternative — CSS Layout

Since <frameset> is deprecated in HTML5, the same multi-section layouts are built today using CSS Grid or Flexbox.

CSS — Modern Grid Layout (replaces frameset) 
/* Define a 3-row, 2-column grid layout */
.page {
  display: grid;
  grid-template-rows: 80px 1fr 50px;
  grid-template-columns: 200px 1fr;
  grid-template-areas:
    "header  header"
    "nav     content"
    "status  status";
  min-height: 100vh;
}

.header  { grid-area: header;  background: #2563eb; color: white; }
.nav     { grid-area: nav;     background: #f1f5f9; }
.content { grid-area: content; padding:  24px;    }
.status  { grid-area: status;  background: #1e293b; color: white; }
🎓
Historical Perspective: Frames taught an important idea — separate navigation from content. Modern Single-Page Apps (SPAs) achieve this elegantly with JavaScript routing, while CSS Grid handles the visual layout.
