XML Introduction

XML stands for Extensible Markup Language. It is designed to store and transport data. Unlike HTML (which focuses on how content is displayed), XML is all about describing what the data means.

Key Features of XML:

  • Self-descriptive: Tags are custom and named by the developer to describe their own data.
  • Platform independent: XML can be read and used across different operating systems, applications, and languages.
  • Human readable: The text-based, tag-wrapped format is easy for anyone to understand.
  • Strict syntax: Every tag that is opened must be properly closed.
💡
HTML vs XML: HTML tells the browser how to display data. XML tells applications what the data is. HTML has predefined tags like <p> and <h1>. In XML, you create your own tags.
🌳

XML Structure

Every XML document has a tree-like structure, starting with a single root element that wraps all other elements. Each element can contain child elements, attributes, and text data.

XML — Student Record 
<?xml version="1.0" encoding="UTF-8"?>
<students>               <!-- Root element -->
  <student>             <!-- Child element -->
    <name>Rahul</name>
    <course>BCA</course>
    <year>3</year>
  </student>
  <student>
    <name>Priya</name>
    <course>MCA</course>
    <year>1</year>
  </student>
</students>

An XML document contains:

  • Root element — the outermost, single parent tag (e.g. <students>)
  • Child elements — nested inside the root (e.g. <student>)
  • Tags — opening and closing labels that wrap data
  • Data values — the actual content between the tags
📜

DTD (Document Type Definition)

A DTD defines the legal building blocks of an XML document. It sets the rules for the structure of the XML — which elements are allowed, how many can appear, and in what order — ensuring all XML files following it have a consistent format.

XML — Internal DTD Example 
<!DOCTYPE students [
  <!ELEMENT students (student+)>  <!-- students contains one or more student -->
  <!ELEMENT student (name, course, year)>
  <!ELEMENT name (#PCDATA)>      <!-- text data -->
  <!ELEMENT course (#PCDATA)>
  <!ELEMENT year (#PCDATA)>
]>
<students>
  <student>
    <name>Rahul</name>
    <course>BCA</course>
    <year>3</year>
  </student>
</students>
🛡️

XML Schema (XSD)

XML Schema (also called XSD — XML Schema Definition) is a more powerful and modern alternative to DTD. It is used to define:

  • Data types — e.g., specifying that <year> must be an integer, not a string
  • Element structure — defining which child elements an element can contain
  • Validation rules — minimum/maximum values, allowed patterns, required fields
⚡
DTD vs XSD: DTD is simpler but limited. XML Schema is verbose but much more precise — it supports data types, namespaces, and complex constraints. XSD is preferred in professional environments.
🎨

XSL / XSLT

XSL (Extensible Stylesheet Language) is a family of languages used to work with XML documents. It has two main parts:

  • XSLT (XSL Transformations) — used to transform XML into other formats such as HTML, plain text, or a different XML structure.
  • XPath — a language for navigating through elements and attributes in an XML document (used inside XSLT).

How XSLT Works:

You give XSLT two files: an XML data file and an XSL template file. The XSLT processor then merges them to produce a final output — typically an HTML page ready for the browser.

XML — Linking an XSL stylesheet 
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/xsl" href="students.xsl"?>
<students>
  <student>
    <name>Rahul</name>
    <course>BCA</course>
  </student>
</students>
⚡

JSON Basics

JSON (JavaScript Object Notation) is a lightweight, text-based format for storing and exchanging structured data. It is the dominant format used today between web servers and client applications (APIs).

Why JSON is Popular:

  • Lightweight: Minimal syntax — no bulky tags like XML.
  • Easy to read: Closely resembles JavaScript objects, familiar and intuitive.
  • Easy to parse: JavaScript can natively convert JSON using JSON.parse().
  • Widely supported: Every modern programming language has a JSON library.
JSON — Basic Structure 
{
  "name": "Rahul",
  "course": "BCA",
  "year": 3,
  "isEnrolled": true
}
🔩

JSON Keys, Values, Arrays & Objects

🔑 Keys & Values

JSON data is written in key / value pairs. The key is always a string wrapped in double quotes. The value can be a string, number, boolean, array, object, or null.

JSON — Keys & Values 
{
  "name": "Rahul",       // string value
  "age": 21,             // number value
  "isEnrolled": true    // boolean value
}

📋 JSON Arrays

A JSON array is an ordered list of values enclosed in square brackets [ ]. It can hold strings, numbers, booleans, objects, or even other arrays.

JSON — Array Example 
{
  "name": "Rahul",
  "subjects": ["HTML", "CSS", "JavaScript", "PHP"]
}

📦 Nested JSON Objects

A JSON value can itself be another JSON object — creating a nested object. This allows complex hierarchical data to be represented.

JSON — Nested Object 
{
  "student": {
    "name": "Rahul",
    "course": "BCA",
    "address": {
      "city": "Mumbai",
      "state": "Maharashtra"
    }
  }
}

Practical — Create XML Student Information File

Create a well-structured XML file to store student data using proper XML syntax.

  • 1 Create a new file called students.xml.
  • 2 Add the XML declaration at the top: <?xml version="1.0" encoding="UTF-8"?>
  • 3 Create a root element <students> and add at least 3 <student> entries inside it.
  • 4 Each student should have: <name>, <course>, <rollno>, and <year> child elements.
  • 5 Open the file in a browser — a well-formed XML will display with the tree shown in its full structure.
💻

Practical — Transform XML Using XSL

Write a basic XSLT transformation to turn your XML student file into a readable HTML table page.

  • 1 Add the XSL stylesheet link to your students.xml using the <?xml-stylesheet?> processing instruction.
  • 2 Create a new file called students.xsl.
  • 3 In the XSL file, use <xsl:template> to define how each <student> should be displayed as an HTML table row.
  • 4 Open the XML file in a browser — the XSLT will kick in and display your data as a formatted HTML table.
💻

Practical — JSON → JavaScript Object

Demonstrate converting JSON-formatted data into a JavaScript object and display it on a webpage.

JavaScript — Parse JSON and Display Data 
// A JSON string (as received from a server or API)
const jsonString = '{"name":"Rahul","course":"BCA","year":3}';

// Convert the JSON string into a JavaScript object
const student = JSON.parse(jsonString);

// Access properties like any normal JS object
console.log(student.name);     // Outputs: Rahul
console.log(student.course);   // Outputs: BCA

// Display it on the page
document.getElementById("output").innerHTML =
  "Name: " + student.name + " | Course: " + student.course;
  • 1 Create a JSON string for a student with name, course, and subjects (as an array).
  • 2 Use JSON.parse() to convert it into a JS object.
  • 3 Display the student name and subjects list inside a <div> element on the webpage using innerHTML.
