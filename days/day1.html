What is the World Wide Web?

The World Wide Web (WWW), commonly called "the web," is a system of interlinked hypertext documents and multimedia content that is accessed via the Internet using web browsers. It was invented by Sir Tim Berners-Lee in 1989 while he was working at CERN (the European physics lab in Switzerland).

The web is built on three core technologies that Tim Berners-Lee defined:

  • HTML — HyperText Markup Language, for creating web pages
  • HTTP — HyperText Transfer Protocol, for transferring pages across the internet
  • URL — Uniform Resource Locator, for addressing pages (web addresses)
💡
Internet ≠ World Wide Web! The Internet is the global physical network of computers connected together (the infrastructure — like roads). The Web is just one service that runs on top of the internet (like cars on those roads). Email, FTP, and streaming services are also separate services that use the internet.

Key Historical Milestones

Year Event
1969 ARPANET (precursor to the Internet) launched by the US Department of Defense
1983 TCP/IP protocol adopted — this is the foundation the modern internet still uses
1989 Tim Berners-Lee proposes the World Wide Web at CERN
1991 The first ever website goes live at info.cern.ch
1993 The Mosaic browser makes the web graphical and accessible to everyone
1994 W3C (World Wide Web Consortium) founded to standardize web technologies
1995 JavaScript invented by Brendan Eich; web boom begins
2004 Web 2.0 era begins — interactive websites, social media, user-generated content

Web Browsers and Web Servers

What is a Web Browser?

A web browser is a software application installed on your computer or phone that lets you access and view web pages. The browser requests pages from servers, receives the HTML/CSS/JS files, and renders (draws) the final visual page you see on screen.

  • 🌐 Google Chrome — most popular (uses Blink engine)
  • 🦊 Mozilla Firefox — open-source, privacy-focused (uses Gecko engine)
  • 🧭 Safari — default on Apple devices (uses WebKit engine)
  • 🌊 Microsoft Edge — modern Windows browser
  • 🎭 Opera — built-in VPN and ad-blocker

What is a Web Server?

A web server is a computer that stores website files and "serves" them to browsers when requested. It runs server software (like Apache or Nginx) that listens for incoming browser requests and responds with the appropriate files.

  • Stores HTML, CSS, JavaScript, images, databases
  • Runs 24/7 to respond to requests from anywhere in the world
  • Has a unique IP address so browsers can find it
✅
Client vs Server: Your computer/phone is the client — it requests pages. The remote computer storing the website is the server — it responds with data. This is called the Client-Server Model and it is the foundation of how the web works.
🔄

HTTP & HTTPS — How Browsers and Servers Communicate

HTTP (HyperText Transfer Protocol) is the set of rules that defines how web browsers and web servers communicate. Think of it as the "language" they both agree to use when exchanging data.

The HTTP Request-Response Cycle

Every time you load a webpage, this sequence happens:

HTTP — Request / Response Cycle 
Step 1: You type https://www.example.com in the browser

Step 2: DNS Lookup
  Browser asks DNS: "What is the IP address for www.example.com?"
  DNS replies:      "It is 93.184.216.34"

Step 3: Browser sends an HTTP GET Request to the server
  GET / HTTP/1.1
  Host: www.example.com
  Accept: text/html

Step 4: Server processes the request and sends back an HTTP Response
  HTTP/1.1 200 OK
  Content-Type: text/html
  <!-- + the full HTML page -->

Step 5: Browser receives HTML, CSS, JS and renders the page

Common HTTP Methods

Method Use
GET Request a page or resource from the server (most common)
POST Submit data to the server (forms, logins)
PUT Update an existing resource on the server
DELETE Delete a resource on the server

Common HTTP Status Codes

Code Meaning
200 OK Request succeeded — page found and returned
301 Moved Permanently Resource has been moved to a new URL permanently
400 Bad Request The request was malformed or invalid
401 Unauthorized Authentication is required to access this resource
403 Forbidden You don't have permission to access this resource
404 Not Found The requested page or file does not exist
500 Internal Server Error Something went wrong on the server's side

HTTP vs HTTPS

HTTPS is the secure version of HTTP. It wraps all communication in SSL/TLS encryption, meaning that even if someone intercepts the traffic between you and the server, they see only scrambled, unreadable data.

🔒
Always look for the padlock 🔒 in your browser's address bar. It confirms the site is using HTTPS and your connection to it is encrypted. Never enter passwords or card details on HTTP-only sites.
📒

DNS — Domain Name System

Computers on the internet communicate using IP addresses (e.g. 142.250.195.14), but humans prefer readable names like google.com. The DNS (Domain Name System) acts like the internet's phone book — it translates human-friendly domain names into IP addresses that computers can route to.

How DNS Works:

  • You type www.google.com in your browser
  • Your browser checks its local cache — has it visited this site before?
  • If not, it asks your ISP's DNS resolver for the IP address
  • The DNS resolver queries the hierarchy of DNS servers until it finds the answer
  • The IP address (142.250.195.14) is returned to your browser
  • Your browser connects directly to that IP and requests the page
💡
DNS is like a phone book: You look up "Google" (the name) and it gives you their phone number (the IP address). Without DNS, you'd need to memorise the IP addresses of every website you want to visit!
🔗

Understanding URLs and Domain Names

A URL (Uniform Resource Locator) is the full web address used to locate a specific resource on the internet. Every URL has distinct parts, each with a specific role.

URL Anatomy 
Protocol    Domain Name           Path         Query String
https://www.example.com/blog/post?id=42&lang=en

┌──────────┐  ┌──────────────────┐  ┌──────────┐  ┌───────────────────┐
│ https:// │  │  www.example.com │  │ /blog/   │  │ ?id=42&lang=en   │
│ Protocol │  │  Domain Name     │  │ post Path│  │ Query Parameters  │
└──────────┘  └──────────────────┘  └──────────┘  └───────────────────┘

Breaking Down the Domain Name

Part Example Description
Subdomain www Optional prefix; "www" is the most common
Second-Level Domain example The unique name registered by the owner
Top-Level Domain (TLD) .com Category: .com (commercial), .org (non-profit), .edu (education), .in (India)
Path /blog/post Points to a specific page or file on the server

Static vs Dynamic Websites

Not all websites work the same way. They fall into two broad categories based on how their content is generated and served.

Static Website Dynamic Website
Content Fixed — same for every visitor Changes based on user, time, data
Technologies HTML, CSS, JavaScript only HTML + Server language (PHP, Node.js) + Database
Speed Faster (no processing needed) Slower (server must generate content)
Examples Portfolio, landing pages, documentation Gmail, Facebook, Amazon, YouTube
In this course ✅ Days 1–14 (HTML, CSS, Bootstrap) ✅ Days 15+ (JavaScript, AJAX, PHP)
