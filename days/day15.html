What is JavaScript?

JavaScript (JS) is a lightweight, interpreted programming language that runs directly in the web browser. It was created in 1995 by Brendan Eich. While HTML provides structure and CSS handles styling, JavaScript is responsible for behaviour — making the page respond, change, and interact with the user.

The Three Layers of a Webpage:

  • 🧱 HTML = Structure (the bones of the page)
  • 🎨 CSS = Presentation (the skin and clothes)
  • ⚡ JavaScript = Behaviour (the muscles and brain)

What can JavaScript do?

  • Validate form input before submission
  • Show/hide content without reloading the page
  • Fetch live data from a server (AJAX)
  • Animate elements and create interactive effects
  • Build full web applications (React, Angular, Vue)
  • Run server-side code with Node.js
💡
JavaScript is NOT Java! Despite similar names, they are completely different languages. Java is a compiled, strongly-typed language used for desktop/Android apps. JavaScript is an interpreted scripting language built for the web.
📎

How to Add JavaScript to HTML

There are three ways to include JavaScript in an HTML document.

1. Inline JavaScript

Written directly inside an HTML event attribute. Not recommended for large code.

HTML — Inline JS 
<button onclick="alert('Hello!')">Click Me</button>

2. Internal JavaScript (using <script> tag)

A <script> block placed inside the HTML file, typically just before the closing </body> tag.

HTML — Internal Script 
<body>
  <h1>My Page</h1>

  <!-- JS placed just before closing body tag -->
  <script>
    alert("Welcome!");
  </script>
</body>

3. External JavaScript (recommended)

JavaScript code in a separate .js file, linked using the src attribute. Best for maintainability and reusability.

HTML — External Script 
<script src="script.js"></script>

Variables — Storing Data

A variable is a named container used to store a piece of data. JavaScript has three keywords to declare variables:

  • var — Old way (function-scoped, can be re-declared). Avoid using it.
  • let — Modern way (block-scoped, can be reassigned). Use for values that change.
  • const — Modern way (block-scoped, cannot be reassigned). Use for values that stay fixed.
JavaScript — Variables 
// Using let — value can change later
let name = "John";
name = "Jane"; // This is allowed

// Using const — value is fixed
const PI = 3.14159;
// PI = 3; // This would cause an ERROR!

// Variables can store many types of data
let age = 25;              // Number
let studentName = "Rahul";  // String
let isEnrolled = true;     // Boolean
🔣

Data Types

JavaScript is a dynamically typed language — you don't need to specify what type of data a variable holds.

JavaScript — Data Types 
// 1. String — text in quotes
let msg = "Hello, World!";

// 2. Number — integers and decimals
let score = 95;
let price = 19.99;

// 3. Boolean — true or false only
let isLoggedIn = false;

// 4. Null — intentionally empty
let data = null;

// 5. Undefined — declared but no value assigned
let result;  // type is "undefined"

// Check the data type using typeof
console.log(typeof msg);       // "string"
console.log(typeof score);     // "number"
console.log(typeof isLoggedIn); // "boolean"

Operators

Arithmetic Operators

JavaScript — Arithmetic Operators 
let a = 10, b = 3;

console.log(a + b);  // 13 — Addition
console.log(a - b);  // 7  — Subtraction
console.log(a * b);  // 30 — Multiplication
console.log(a / b);  // 3.33 — Division
console.log(a % b);  // 1 — Modulus (remainder)
console.log(a ** b); // 1000 — Exponentiation (10³)

Comparison Operators

JavaScript — Comparison Operators 
console.log(5 == "5");  // true  — equal value (loose)
console.log(5 === "5"); // false — equal value AND type (strict)
console.log(10 !== 5); // true  — not equal
console.log(10 > 5);   // true  — greater than
console.log(10 < 5);   // false — less than
console.log(10 >= 10); // true  — greater than or equal
💡
Always use === instead of == in your code. The triple equals checks both value AND type, preventing many common bugs. For example, 0 == false is true but 0 === false is false.
🗂️

Arrays

An array is a special variable that can hold more than one value at a time. It stores an ordered list of items inside square brackets [ ]. Array indexing starts at 0, not 1.

JavaScript — Arrays 
// Declaring an array
let fruits = ["Apple", "Banana", "Mango"];

// Accessing items by index (starts at 0)
console.log(fruits[0]); // "Apple"
console.log(fruits[2]); // "Mango"

// Finding the number of items
console.log(fruits.length); // 3

// Adding an item to the end
fruits.push("Orange");
console.log(fruits); // ["Apple", "Banana", "Mango", "Orange"]

// Removing the last item
fruits.pop();
console.log(fruits); // ["Apple", "Banana", "Mango"]

// Arrays can hold mixed data types
let mixed = ["Rahul", 21, true, null];
🖥️

Output Systems

JavaScript provides multiple ways to display output or communicate with the user.

JavaScript — All Output Methods 
// 1. alert() — Shows a popup message box to the user
alert("Hello! Welcome to JavaScript.");

// 2. console.log() — Prints to the developer console (F12)
let score = 95;
console.log("Your score is:", score); // Your score is: 95

// 3. prompt() — Gets text input from the user via a popup
let userName = prompt("What is your name?");
alert("Nice to meet you, " + userName + "!");

// 4. document.write() — Writes text directly into the HTML page
document.write("<h2>Page generated by JavaScript!</h2>");

// 5. innerHTML — Updates the content of an existing HTML element
document.getElementById("output").innerHTML = "Hello from JS!";

Quick Comparison Table:

Method Use Case
alert() Show a simple popup message to the user
console.log() Debugging — visible only in browser DevTools (F12)
prompt() Collect a short text input from the user
document.write() Write content to the page (mostly for testing)
innerHTML Update content inside a specific HTML element
