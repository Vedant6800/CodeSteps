Responsive Web Design

Responsive Web Design (RWD) means building webpages that automatically adapt their layout and appearance to fit any screen size — whether viewed on a mobile phone, tablet, or desktop monitor.

Why is Responsive Design Important?

  • Over 60% of web traffic comes from mobile devices
  • A responsive site provides a good experience on every device
  • Google ranks mobile-friendly websites higher in search results

Step 1 — The Viewport Meta Tag

Always add this to your <head> — it tells the browser how to control the page dimensions on mobile devices.

HTML — Viewport Meta Tag 
<meta name="viewport"
      content="width=device-width, initial-scale=1.0">

Step 2 — CSS Media Queries

Media queries apply different CSS rules depending on the screen width. This is the core technique of responsive design.

CSS — Media Queries 
/* Default styles (Desktop first) */
.container {
  display: flex;
  gap: 24px;
}

/* Tablet (max-width 768px) */
@media (max-width: 768px) {
  .container {
    flex-direction: column;
  }
  nav ul {
    display: none; /* hide nav links on small screens */
  }
}

/* Mobile (max-width 480px) */
@media (max-width: 480px) {
  body {
    font-size: 14px;
  }
  h1 {
    font-size: 1.5rem;
  }
}
💡
Mobile First: A popular modern approach is to write styles for mobile first, then use min-width media queries to add styles for larger screens. This keeps the base CSS lean.
📦

<div> as a Layout Container

The <div> (Division) element is the most versatile layout tool in HTML. It is a block-level container with no visual styling of its own — you apply CSS to make it a header, card, column, or any other component.

In responsive design, <div> elements are grouped and rearranged using Flexbox or Grid to change from a multi-column desktop layout to a single-column mobile layout.

HTML + CSS — Responsive div columns 
<!-- HTML -->
<div class="row">
  <div class="col">Column 1</div>
  <div class="col">Column 2</div>
  <div class="col">Column 3</div>
</div>

/* CSS */
.row { display: flex; gap: 16px; }
.col { flex: 1; padding: 16px; background: #f0f4f8; }

@media (max-width: 600px) {
  .row { flex-direction: column; }
}

Navbar Design

A navigation bar (navbar) is placed at the top of a webpage and contains links to different pages or sections. It is one of the most important UI components on any website.

HTML + CSS — Navbar 
<!-- HTML -->
<nav class="navbar">
  <div class="brand">🎓 MyCollege</div>
  <ul class="nav-links">
    <li><a href="#Home">Home</a></li>
    <li><a href="#about">About</a></li>
    <li><a href="#courses">Courses</a></li>
    <li><a href="#contact">Contact</a></li>
  </ul>
</nav>

/* CSS */
.navbar {
  display: flex;
  justify-content: space-between;
  align-items: center;
  background: #1e293b;
  padding: 14px 24px;
}
.brand { color: white; font-size: 1.3rem; font-weight: 700; }
.nav-links { list-style: none; display: flex; gap: 24px; }
.nav-links a {
  color: #cbd5e1;
  text-decoration: none;
  transition: color 0.2s;
}
.nav-links a:hover { color: white; }

Practical 8 — College Web Page Using All CSS Types

Build a College Website that demonstrates all three CSS types — Inline, Internal, and External. The page should include:

  • 🧭 Navbar — college name logo and navigation links (Home, About, Courses, Contact)
  • 🏷️ Hero/Header section — styled with External CSS, gradient background
  • 📚 Courses section — three course cards using a responsive flex layout
  • 📞 Contact section — name, email, phone
  • 🔻 Footer — copyright info
  • 📱 Responsive — cards stack vertically on small screens
📂
Create college.html + college.css. Use External CSS for main layout, Internal CSS for page-specific tweaks, and Inline CSS for individual element overrides.
