Functions

A function is a reusable block of code designed to perform a specific task. Instead of writing the same code repeatedly, you write it once in a function and "call" it whenever you need it.

Defining and Calling a Function

JavaScript — Basic Function 
// Defining the function using the "function" keyword
function greet() {
  alert("Welcome to our website!");
}

// Calling (invoking) the function
greet();   // The alert popup will now appear
greet();   // Can be called again and again

Functions with Parameters

A parameter is a placeholder variable listed inside the brackets when defining a function. An argument is the actual value you pass in when calling it.

JavaScript — Parameters & Arguments 
// "name" and "age" are PARAMETERS (placeholders)
function introduce(name, age) {
  alert("Hi, I am " + name + " and I am " + age + " years old.");
}

// "Rahul" and 21 are ARGUMENTS (real values)
introduce("Rahul", 21);  // Hi, I am Rahul and I am 21 years old.
introduce("Priya", 19);  // Hi, I am Priya and I am 19 years old.

Return Values

A function can use the return keyword to send a value back to wherever it was called from. This allows functions to calculate something and pass the result back.

JavaScript — Return Values 
function add(a, b) {
  return a + b;  // sends the result back
}

let total = add(10, 5);  // total = 15
console.log("Sum is:", total);  // Sum is: 15

function isAdult(age) {
  return age >= 18;  // returns true or false
}
console.log(isAdult(20));  // true
console.log(isAdult(15));  // false

Arrow Functions (Modern ES6 Syntax)

Arrow functions are a shorter, modern way to write functions. They use => instead of the function keyword.

JavaScript — Arrow Functions 
// Traditional function
function multiply(a, b) {
  return a * b;
}

// Same function written as an Arrow Function
const multiply = (a, b) => a * b;

console.log(multiply(4, 5)); // 20

Conditional Statements

Conditional statements allow your program to make decisions and execute different code depending on whether a condition is true or false.

JavaScript — if / else if / else 
let marks = 75;

if (marks >= 90) {
  console.log("Grade: A — Distinction");
} else if (marks >= 75) {
  console.log("Grade: B — First Class");
} else if (marks >= 60) {
  console.log("Grade: C — Second Class");
} else {
  console.log("Grade: F — Fail");
}
// Output: "Grade: B — First Class" (since marks is 75)

HTML Events

Events are actions or occurrences that happen in the browser, usually triggered by the user. JavaScript can respond to these events and run specific code when they occur.

Common HTML Events:

Event Triggered When…
onclick User clicks a button or element
onmouseover Mouse pointer moves over an element
onmouseout Mouse pointer moves away from an element
onchange The value of an input field changes
onkeyup User releases a key on the keyboard
onload Page (or image) has fully loaded
onsubmit Form is submitted

Example — Events in Action

HTML + JS — Multiple Events Demo 
<!DOCTYPE html>
<html>
<body>

  <!-- onclick event -->
  <button onclick="greet()">Click Me</button>

  <!-- onmouseover and onmouseout events -->
  <div
    onmouseover="this.style.background='yellow'"
    onmouseout="this.style.background='white'"
    style="padding:20px;border:1px solid #ccc;">
    Hover over me!
  </div>

  <!-- onkeyup event —tracks typing in real-time -->
  <input type="text" onkeyup="showInput(this.value)" placeholder="Type here...">
  <p id="typed"></p>

  <!-- onchange event -->
  <select onchange="alert('You selected: ' + this.value)">
    <option>HTML</option>
    <option>CSS</option>
    <option>JavaScript</option>
  </select>

  <script>
    function greet() {
      alert("You clicked the button!");
    }
    function showInput(val) {
      document.getElementById("typed").innerHTML = "You typed: " + val;
    }
  </script>
</body>
</html>

DOM — Document Object Model

When a browser loads an HTML page, it creates a DOM (Document Object Model) — a tree-like representation of every element in the document. JavaScript uses the DOM to access, change, add, or remove HTML elements and their styles dynamically without reloading the page.

How the DOM Tree looks:

DOM Tree Structure 
document
  └── html
        ├── head
        │     └── title → "My Page"
        └── body
              ├── h1 → "Welcome"
              ├── p  → "Hello World"
              └── button → "Click Me"

Selecting Elements

JavaScript — DOM Selection Methods 
// 1. By ID — most specific, only one element
let heading = document.getElementById("main-heading");

// 2. By Class Name — returns a collection of elements
let cards = document.getElementsByClassName("card");

// 3. By Tag Name — gets all elements of that tag
let paragraphs = document.getElementsByTagName("p");

// 4. querySelector — CSS selector, returns first match
let btn = document.querySelector("#submit-btn");

// 5. querySelectorAll — CSS selector, returns all matches
let allCards = document.querySelectorAll(".card");

Modifying Elements

JavaScript — DOM Manipulation 
let el = document.getElementById("my-div");

// Change the text content
el.innerHTML = "<strong>Updated!</strong>";
el.textContent = "Just plain text, no HTML";

// Change CSS styles directly
el.style.color = "red";
el.style.backgroundColor = "#f0f4f8";
el.style.fontSize = "24px";

// Add or remove a CSS class
el.classList.add("active");
el.classList.remove("hidden");
el.classList.toggle("dark-mode");
💡
innerHTML vs textContent: Use innerHTML when you want to insert HTML tags (like <strong>). Use textContent when you only want plain text — it's safer as it prevents HTML injection attacks.

Practical 10 — JavaScript Program Using 5 Events

Build a single HTML page that demonstrates at least 5 different JavaScript events. Each event should produce a visible result on the page (not just an alert).

  • 1 Add a button that uses onclick to change the background colour of a <div> using style.backgroundColor.
  • 2 Create a box that changes colour on onmouseover and resets on onmouseout.
  • 3 Create a text input with onkeyup that displays everything the user types inside a <p> element in real-time.
  • 4 Create a <select> dropdown that uses onchange to update a paragraph showing which option was selected.
  • 5 Write a function that validates that a name input is not empty when a button is clicked. If empty, display an error message in red.
  • 6 Bonus: Use querySelector (instead of getElementById) for at least one of the DOM selections above.
